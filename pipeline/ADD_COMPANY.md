# 企業を1社追加する手順書（パイプライン）

**対象読者**：共同編集メンバー（どちらのPCでも同じ手順で追加できます）
**所要時間の目安**：AIに任せる場合 約10分／手動の場合 約40分

---

## 全体像（5ステップ）

```
① リサーチ ──→ ② data.js に追記 ──→ ③ ロゴ取得 ──→ ④ 自動検証 ──→ ⑤ 監修・共有
   公開情報を       テンプレを埋めて       スクリプト        _データ検証.html    2人でチェックリスト
   集める           末尾に貼る             1コマンド          エラー0を確認       確認→CHANGELOG追記
```

**いちばんラクな方法**：Claude Code（またはお使いのAIアシスタント）に次のように頼むと、①〜④をまとめて実行できます。

> 就活辞典に「◯◯株式会社」を追加してください。`pipeline/ADD_COMPANY.md` の手順に従い、
> リサーチ→data.js追記→ロゴ取得→検証まで行って、最後に監修用の出典一覧を出してください。

以下は、その中で何が行われるか（＝手動でやる場合の手順）です。

---

## STEP 1　リサーチ（公開情報を集める）

**使う出典（信頼度順）**
1. 有価証券報告書・決算資料（売上・資本金・従業員数・平均年収・平均年齢）
2. 企業の採用サイト（初任給・休日・福利厚生・求める人物像・選考フロー・インターン）
3. 就職情報サイト（リクナビ/マイナビ等：説明会・募集職種の裏取り）
4. 口コミサイト（OpenWork等：働き方の**傾向**の把握のみ。数値は使わない）

**AIに依頼する場合のリサーチプロンプト（コピペ用）**

```
「◯◯株式会社」について、就活生向け企業辞典に載せるデータを公開情報から調べてください。

必要な項目：
- 正式社名・読みがな(ひらがな)・英文社名・証券コード(非上場なら「非上場」)
- 公式サイトのURL(https://始まりのコーポレートサイト)
- 業種・本社所在地(住所番地まで)・設立年・上場区分
- 資本金・従業員数(連結)・売上高(直近通期)・平均年収・平均年齢
- 大卒/院卒の初任給(月給)・年間休日・有給取得率・女性比率・月平均残業時間
- 会社のスローガンやキャッチコピー(実際のもの)
- 事業内容の説明(2〜4文)・同業他社との違い・働き方の良い点と注意点(両方)
- 募集職種・福利厚生・インターンの有無と内容・説明会の形式と時期・選考フロー
- 採用サイトに書かれた「求める人物像」を3つに要約

条件：
- 各数値には出典(どの資料の何年版か)を添えること
- 見つからない項目は「不明」と正直に書くこと(推測で埋めない)
- 口コミサイトの情報は「傾向」としてのみ扱うこと
```

---

## STEP 2　data.js に追記する

1. `data.js` を開き、`DATA` 配列の**最後の会社**（`}` で終わる行）の直後に `,` を付ける
2. `SCHEMA.md` §6 のテンプレートをコピーして貼り付け、リサーチ結果で埋める

**埋めるときの要点**
- `no` … 既存の最大値 +1（欠番・変更は禁止）
- `stats`（能力値6軸）… `SCHEMA.md` §3 のルーブリックに従い、**既存の似た会社と見比べて**付ける
- `rel` … 同業や比較されやすい会社の `no` を3つ。**逆方向も検討**（既存企業の `rel` に新会社を入れると相互に行き来できる）
- `point` … 良い点だけでなく**注意点も必ず**書く
- 語彙項目（`cat`/`region`/`jobs`/`welfare`）… 必ず既存リストの言葉をそのまま使う

---

## STEP 3　ロゴを取得する

**自動（推奨）**：PowerShell を開き、このフォルダで次を実行

```powershell
cd pipeline
.\fetch_logo.ps1 -Title "Hitachi" -No 21 -Slug "hitachi"
```

- `-Title` … **英語版Wikipediaの記事名**（例: トヨタ→ `"Toyota"`、日立→ `"Hitachi"`。まず en.wikipedia.org で記事を検索して正確な題名を確認）
- `-No` … 図鑑No（data.js に書いた `no` と同じ数字）
- `-Slug` … 英小文字の識別名（ファイル名になる。例: `hitachi`）
- 成功すると `assets/logos/21_hitachi.png` が保存される → data.js の `logo:"21_hitachi"` と一致させる

**必ず目視確認**：保存されたPNGを開き、①正しい会社か ②現行ロゴか ③ロゴでなく社屋写真などになっていないか を確認。
ダメな場合は英語版Wikipediaの記事内のロゴ画像を手動で保存（512px程度）して同じ名前で置く。

> ⚠ **権利の注意**：実在ロゴの使用は**社内プロトタイプ用途に限定**。外部公開時は各社の商標ガイドライン確認が必要（README参照）。

---

## STEP 4　自動検証

`_データ検証.html` をダブルクリック → **「✓ 全チェック合格」** になるまで修正。
（△の注意は、内容を確認して意図的であればそのままで可）

あわせて `index.html` を開いて、新しい会社が一覧・詳細・くらべる・ランキングに出ることを目で確認。

---

## STEP 5　監修・共有

1. `REVIEW_CHECKLIST.md` を使って**もう一人が**チェック（作った人と別の人が見る）
2. `CHANGELOG.md` に1行追記（例：`2026-07-05 No.21 日立製作所を追加（担当:宮崎、監修:◯◯）`）
3. データを見直した場合は `data.js` 冒頭の `DATA_ASOF` と `NOTIFS`（お知らせ）も更新
4. フォルダごと共有（このフォルダは自己完結なので、zip や Drive でそのまま渡せます）

## よくあるつまずき

| 症状 | 原因と対処 |
|---|---|
| アプリに新しい会社が出ない | data.js の構文エラー（`,` の付け忘れが大半）。検証ページを開くと分かる |
| ロゴが頭文字の丸になる | `logo` の値と PNG ファイル名の不一致。検証ページの「ロゴ画像が見つかりません」を確認 |
| 検証ページで「語彙にありません」 | `cat`/`region`/`jobs`/`welfare` の言葉が既存リストと1字でも違う。SCHEMA.md §1 のリストからコピペする |
| 文字化けする | data.js は必ず **UTF-8** で保存（メモ帳なら「名前を付けて保存→エンコード UTF-8」） |
| ロゴ取得で「429 / Too many requests」 | Wikipediaの一時的なアクセス制限。スクリプトは自動で3回リトライするが、それでもダメなら**数分〜数十分おいて再実行**。急ぐ場合はブラウザで英語版Wikipedia記事のロゴ画像を開き、右クリック保存→`NN_slug.png`に改名して `assets/logos/` へ |
| ロゴがビル写真など「ロゴ以外」になる | 記事の先頭画像が写真のケース。スクリプトは「logo」を含むファイル名を優先的に探すが、見つからないと先頭画像で代用する。手動保存に切り替える |
