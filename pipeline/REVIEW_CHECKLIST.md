# 監修チェックリスト（企業追加・データ更新のたびに使う）

**ルール**：データを作った人とは**別の人**がチェックする（自分の作ったものは自分で監修しない）。
全部 ✓ になったら共有OK。1つでも直したら再チェック。

対象企業：No.＿＿＿ 社名＿＿＿＿＿＿＿＿＿＿　作成：＿＿＿＿　監修：＿＿＿＿　日付：＿＿＿＿

---

## A. 機械チェック（1分）

- [ ] `_データ検証.html` を開いて **エラー0件**（△注意は内容確認済み）
- [ ] `index.html` を開いて、一覧カード・詳細・くらべる・ランキング・五十音索引に新しい会社が出る
- [ ] ロゴが正しく表示される（頭文字の丸になっていない）

## B. 事実チェック（5分）

- [ ] 売上・従業員数・平均年収・資本金が**出典と一致**（有報・IR資料の何年版かを確認）
- [ ] 初任給・年間休日・福利厚生が**採用サイトの現行情報**と一致
- [ ] 証券コード・上場区分・本社住所が正しい
- [ ] 設立年の表示（founded）と数値（foundedY）が一致
- [ ] インターン・説明会の情報が今シーズンのもの

## C. 編集品質チェック（5分）

- [ ] `summary`／`feature` が**その会社にしか当てはまらない**内容になっている（どの会社にも言える一般論はNG）
- [ ] `point` に**注意点も**書かれている（良い話だけになっていない）
- [ ] `catch` は実際のスローガン（創作なら創作と分かる範囲）
- [ ] `stats` 6軸が既存の似た会社と見比べて違和感ない（SCHEMA.md §3のアンカー参照）
- [ ] `rel`（あわせて見たい）が納得感のある3社。逆方向（既存企業→新会社）も検討したか
- [ ] 誹謗・断定しすぎ・古い情報がない

## D. 権利・表記チェック（2分）

- [ ] ロゴは**現行デザイン**で、正しい会社のもの
- [ ] ロゴの用途が社内プロトタイプの範囲に収まっている（外部公開するときは要権利確認）
- [ ] `official:true`（企業公認フラグ）は、**企業の担当者に直接確認できた場合のみ**付いている

## E. 記録（1分）

- [ ] `CHANGELOG.md` に1行追記した（日付・No・社名・担当・監修）
- [ ] データ基準を見直した場合は `DATA_ASOF` と `NOTIFS` も更新した
